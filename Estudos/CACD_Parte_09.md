# **Comprehensive Guide to Adversarial Attacks: FGSM, C&W, and PGD**

## **Introduction: What Are Adversarial Attacks?**

Adversarial attacks exploit machine learning models by adding carefully crafted, imperceptible perturbations to inputs that cause the model to make incorrect predictions.

**Core Concept:**
```
Clean Image + Tiny Perturbation = Adversarial Example
Panda (99.3% confidence) + Noise (invisible to human) = Gibbon (99.7% confidence)
```

**Key Properties:**
- **Imperceptibility**: Humans can't see the difference
- **Transferability**: Adversarial examples often work across different models
- **Digital vs Physical**: Some attacks work in real world (printed, 3D objects)

---

# **1. FGSM - Fast Gradient Sign Method**

## **Overview**

**FGSM** is the simplest and fastest adversarial attack method, introduced by Goodfellow et al. (2014). It's a **single-step attack** that uses the model's gradients to craft adversarial examples.

**Key Characteristics:**
- **Speed**: Very fast (one gradient computation)
- **Power**: Moderate effectiveness
- **Simplicity**: Easy to understand and implement
- **Use Case**: Quick baseline attack, adversarial training

---

## **Mathematical Foundation**

### **The Goal:**
Maximize the model's loss function to cause misclassification.

### **The Formula:**

```
x_adv = x + ε × sign(∇_x J(θ, x, y))

Where:
- x = original input (clean image)
- x_adv = adversarial example
- ε (epsilon) = perturbation magnitude (controls visibility)
- J = loss function (e.g., cross-entropy)
- θ = model parameters
- y = true label
- ∇_x = gradient with respect to input
- sign() = sign function (-1, 0, or +1)
```

### **Breaking It Down:**

**Step 1: Compute Gradient**
```
∇_x J(θ, x, y) = how to change each pixel to increase loss
```
This gradient tells us which direction to modify each pixel to make the model more wrong.

**Step 2: Take the Sign**
```
sign(gradient) = direction of change (positive or negative)
```
We only care about the direction, not the magnitude.

**Step 3: Add Perturbation**
```
x_adv = x + ε × sign(gradient)
```
Move each pixel by ε in the direction that increases loss.

---

## **How FGSM Works: Step-by-Step**

### **Example: Attacking an Image Classifier**

**Starting Point:**
- **Input**: Image of a panda (224×224×3 pixels)
- **Model**: CNN trained on ImageNet
- **Current Prediction**: "Panda" (99.3% confidence)
- **Goal**: Make model predict "Gibbon"

**Process:**

```
Step 1: Forward Pass
-------
Input: panda image → Model → Output: [0.993 for Panda, 0.001 for Gibbon, ...]
Loss = CrossEntropy(prediction, true_label="Panda") = 0.007 (low loss = good prediction)

Step 2: Compute Gradient
-------
∂Loss/∂pixel[i,j,k] = how much does changing this pixel change the loss?

Example gradient values (simplified):
pixel[0,0,0]: +0.0023
pixel[0,0,1]: -0.0015
pixel[0,0,2]: +0.0031
...

Step 3: Apply Sign Function
-------
sign(+0.0023) = +1
sign(-0.0015) = -1
sign(+0.0031) = +1
...

Step 4: Create Perturbation
-------
If ε = 0.01 (8/255 in [0-255] range):
perturbation[0,0,0] = 0.01 × (+1) = +0.01
perturbation[0,0,1] = 0.01 × (-1) = -0.01
perturbation[0,0,2] = 0.01 × (+1) = +0.01
...

Step 5: Generate Adversarial Example
-------
x_adv = x + perturbation
adversarial_panda = original_panda + noise

Step 6: Test Attack
-------
Input: adversarial_panda → Model → Output: [0.002 for Panda, 0.997 for Gibbon, ...]
Success! Model now predicts "Gibbon" with 99.7% confidence
```

---

## **FGSM: Code Implementation**

### **PyTorch Implementation:**

```python
import torch
import torch.nn as nn

def fgsm_attack(image, epsilon, data_grad):
    """
    FGSM attack implementation
    
    Args:
        image: Original input image (tensor)
        epsilon: Perturbation magnitude (float)
        data_grad: Gradient of loss w.r.t. input (tensor)
    
    Returns:
        Adversarial image (tensor)
    """
    # Get the sign of the gradient
    sign_data_grad = data_grad.sign()
    
    # Create the perturbation
    perturbation = epsilon * sign_data_grad
    
    # Add perturbation to original image
    perturbed_image = image + perturbation
    
    # Clip to maintain valid pixel range [0, 1]
    perturbed_image = torch.clamp(perturbed_image, 0, 1)
    
    return perturbed_image

# Full attack example
def generate_adversarial_example(model, image, label, epsilon):
    """
    Generate adversarial example using FGSM
    """
    # Set requires_grad to True to compute gradients w.r.t. input
    image.requires_grad = True
    
    # Forward pass
    output = model(image)
    
    # Calculate loss
    criterion = nn.CrossEntropyLoss()
    loss = criterion(output, label)
    
    # Zero all existing gradients
    model.zero_grad()
    
    # Backward pass to compute gradients
    loss.backward()
    
    # Get gradient of input
    data_grad = image.grad.data
    
    # Generate adversarial example
    perturbed_image = fgsm_attack(image, epsilon, data_grad)
    
    return perturbed_image

# Usage example
epsilon = 8/255  # Common value (approximately 3% perturbation)
adversarial_image = generate_adversarial_example(model, clean_image, true_label, epsilon)

# Test the attack
original_pred = model(clean_image).argmax()
adversarial_pred = model(adversarial_image).argmax()

print(f"Original prediction: {original_pred}")
print(f"Adversarial prediction: {adversarial_pred}")
```

## **FGSM Variants**

### **1. Targeted FGSM**
Goal: Force model to predict a **specific** target class

```python
def targeted_fgsm(image, epsilon, data_grad):
    """
    Targeted FGSM: minimize loss for target class
    """
    # Note the MINUS sign (we want to minimize loss for target)
    sign_data_grad = data_grad.sign()
    perturbed_image = image - epsilon * sign_data_grad
    return torch.clamp(perturbed_image, 0, 1)

# Usage: compute loss with respect to TARGET label
loss = criterion(output, target_label)  # Not true label!
```

### **2. Iterative FGSM (I-FGSM / BIM - Basic Iterative Method)**
Apply FGSM multiple times with smaller steps

```python
def iterative_fgsm(image, epsilon, alpha, num_iter, model, label):
    """
    I-FGSM: Multiple small FGSM steps
    
    Args:
        alpha: Step size per iteration (typically epsilon/num_iter)
        num_iter: Number of iterations
    """
    perturbed_image = image.clone()
    
    for i in range(num_iter):
        perturbed_image.requires_grad = True
        output = model(perturbed_image)
        loss = nn.CrossEntropyLoss()(output, label)
        
        model.zero_grad()
        loss.backward()
        data_grad = perturbed_image.grad.data
        
        # FGSM step
        perturbed_image = perturbed_image + alpha * data_grad.sign()
        
        # Project back to epsilon ball around original image
        perturbation = torch.clamp(perturbed_image - image, -epsilon, epsilon)
        perturbed_image = torch.clamp(image + perturbation, 0, 1).detach()
    
    return perturbed_image

# Usage
epsilon = 8/255
alpha = epsilon/10  # Smaller steps
num_iter = 10
adversarial = iterative_fgsm(image, epsilon, alpha, num_iter, model, label)
```

## **FGSM: Strengths and Weaknesses**

### **Strengths:**
✓ **Very fast**: Single gradient computation  
✓ **Simple to implement**: Few lines of code  
✓ **Good for adversarial training**: Fast generation of training examples  
✓ **Baseline attack**: Standard comparison point  

### **Weaknesses:**
✗ **Lower success rate**: Less effective than iterative methods  
✗ **Large perturbations**: May need higher ε to succeed  
✗ **Not optimal**: Single step doesn't find best perturbation  
✗ **Gradient masking vulnerable**: Adaptive defenses can counter it

# **2. C&W Attack - Carlini & Wagner**

## **Overview**

The **Carlini & Wagner (C&W) attack** (2017) is considered one of the **most powerful adversarial attacks**. It's an optimization-based attack that finds minimal perturbations through iterative refinement.

**Key Characteristics:**
- **Power**: Extremely effective (near 100% success rate)
- **Stealth**: Produces minimal, imperceptible perturbations
- **Complexity**: Computationally expensive
- **Robustness**: Works even against defensive distillation

**Key Insight:** Frame the attack as an optimization problem with careful objective function design.

---

## **Mathematical Foundation**

### **The Challenge:**

We want to solve:
```
minimize ||δ||_p subject to:
1. C(x + δ) = t (misclassification constraint)
2. x + δ ∈ [0, 1]^n (valid pixel range)

Where:
- δ = perturbation
- ||δ||_p = perturbation size (L0, L2, or L∞ norm)
- C(x) = classifier function
- t = target class (for targeted attack)
```

**Problem**: This is hard to optimize directly (constraints are non-differentiable).

### **C&W's Solution:**

Convert to unconstrained optimization:

```
minimize c × f(x + δ) + ||δ||_p

Where f(x + δ) is a loss function that encourages misclassification
```

---

## **The C&W Objective Function**

### **Core Innovation: The f() Function**

Instead of using standard cross-entropy loss, C&W designed a custom function:

```
f(x') = max(max{Z(x')_i : i ≠ t} - Z(x')_t, -κ)

Where:
- Z(x') = logits (pre-softmax outputs)
- t = target class
- κ (kappa) = confidence parameter
- max{Z(x')_i : i ≠ t} = highest logit among non-target classes
```

### **Understanding f():**

**What it does:**
- **When attack fails**: f(x') is large and positive
- **When attack barely succeeds**: f(x') ≈ 0
- **When attack succeeds with high confidence**: f(x') is negative

**The κ parameter:**
```
κ = 0: Just barely misclassify
κ = 5: Misclassify with 5-unit logit margin (higher confidence)
```

### **Full C&W L2 Objective:**

```
minimize ||δ||_2² + c × f(x + δ)

subject to: x + δ ∈ [0, 1]^n
```

**Balancing act:**
- **||δ||_2²** term: Keep perturbation small
- **c × f(x + δ)** term: Ensure misclassification
- **c** (constant): Controls trade-off

---

## **How C&W Works: Step-by-Step**

### **The Algorithm:**

```
Input: 
- Original image x
- Target class t
- Model C

Step 1: Change of Variables (Handle Box Constraints)
-------
Instead of optimizing δ directly (with [0,1] constraints),
optimize w in unconstrained space:

x_adv = 0.5 × (tanh(w) + 1)

This ensures x_adv ∈ [0, 1] automatically!

Step 2: Binary Search for c
-------
We need to find the smallest perturbation that still causes misclassification.

Initialize: c_min = 0, c_max = large value
for each binary search iteration:
    c = (c_min + c_max) / 2
    
    Run optimization with this c value →
    
    if attack succeeds:
        c_max = c  (try smaller c)
        Save this solution
    else:
        c_min = c  (need larger c)

Step 3: Optimization for Fixed c
-------
For current c value, minimize:
    L(w) = ||0.5(tanh(w)+1) - x||_2² + c × f(0.5(tanh(w)+1))

Using gradient descent:
for iteration in range(max_iterations):
    # Compute gradient
    ∇L = ∂L/∂w
    
    # Update w
    w = w - learning_rate × ∇L
    
    # Check if attack succeeded
    if f(x_adv) < 0:
        attack_successful = True

Step 4: Return Best Solution
-------
Return adversarial example with smallest ||δ||_2 that succeeded
```

## **C&W: Code Implementation**

### **PyTorch Implementation (L2 Attack):**

```python
import torch
import torch.nn as nn
import torch.optim as optim

class CWL2Attack:
    def __init__(self, model, targeted=True, c=1.0, kappa=0, 
                 max_iter=1000, learning_rate=0.01, binary_search_steps=9):
        """
        Carlini & Wagner L2 Attack
        
        Args:
            model: Target model
            targeted: Targeted attack if True
            c: Initial value of c constant
            kappa: Confidence parameter
            max_iter: Maximum optimization iterations
            learning_rate: Learning rate for Adam optimizer
            binary_search_steps: Number of binary search steps for c
        """
        self.model = model
        self.targeted = targeted
        self.c = c
        self.kappa = kappa
        self.max_iter = max_iter
        self.learning_rate = learning_rate
        self.binary_search_steps = binary_search_steps
    
    def _f(self, outputs, labels):
        """
        C&W's f function
        
        Returns:
            Loss that encourages misclassification
        """
        one_hot_labels = torch.eye(outputs.shape[1])[labels].to(outputs.device)
        
        # Get logit for target class
        real = (one_hot_labels * outputs).sum(dim=1)
        
        # Get maximum logit for other classes
        other = ((1 - one_hot_labels) * outputs - one_hot_labels * 10000).max(dim=1)[0]
        
        if self.targeted:
            # Targeted: want target class logit > others
            loss = torch.clamp(other - real + self.kappa, min=0)
        else:
            # Untargeted: want other class logit > target
            loss = torch.clamp(real - other + self.kappa, min=0)
        
        return loss.sum()
    
    def _optimize(self, images, labels, c):
        """
        Optimize for fixed c value
        """
        batch_size = images.shape[0]
        
        # Change of variables: w = arctanh(2x - 1)
        # This ensures x_adv ∈ [0, 1] when x_adv = 0.5(tanh(w) + 1)
        w = torch.zeros_like(images, requires_grad=True)
        w.data = torch.atanh((images * 2 - 1) * 0.999)  # 0.999 to avoid inf
        
        # Optimizer
        optimizer = optim.Adam([w], lr=self.learning_rate)
        
        best_perturbation = None
        best_l2 = float('inf')
        
        for step in range(self.max_iter):
            # Generate adversarial example from w
            x_adv = 0.5 * (torch.tanh(w) + 1)
            
            # Forward pass
            outputs = self.model(x_adv)
            
            # Compute L2 distance
            l2_distance = torch.norm((x_adv - images).view(batch_size, -1), p=2, dim=1)
            
            # Compute f() loss
            f_loss = self._f(outputs, labels)
            
            # Total loss
            loss = l2_distance.sum() + c * f_loss
            
            # Optimize
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            
            # Check if attack succeeded with better perturbation
            pred_labels = outputs.argmax(dim=1)
            if self.targeted:
                success = (pred_labels == labels)
            else:
                success = (pred_labels != labels)
            
            for i in range(batch_size):
                if success[i] and l2_distance[i] < best_l2:
                    best_l2 = l2_distance[i].item()
                    best_perturbation = x_adv[i].detach().clone()
        
        return best_perturbation, best_l2
    
    def attack(self, images, labels):
        """
        Perform C&W attack with binary search
        """
        batch_size = images.shape[0]
        
        # Binary search bounds for c
        c_min = 0
        c_max = 1.0
        
        best_adversarial = images.clone()
        best_l2 = float('inf')
        
        for binary_step in range(self.binary_search_steps):
            # Current c value
            current_c = (c_min + c_max) / 2
            
            # Optimize with current c
            adv_example, l2_dist = self._optimize(images, labels, current_c)
            
            if adv_example is not None:
                # Attack succeeded
                best_adversarial = adv_example
                best_l2 = l2_dist
                c_max = current_c  # Try smaller c
            else:
                # Attack failed
                c_min = current_c  # Need larger c
        
        return best_adversarial

# Usage example
model.eval()
attacker = CWL2Attack(
    model=model, 
    targeted=True, 
    c=1.0, 
    kappa=0,
    max_iter=1000,
    learning_rate=0.01,
    binary_search_steps=9
)

# Generate adversarial example
clean_image = ...  # Shape: [1, 3, 224, 224]
target_label = torch.tensor([TARGET_CLASS])  # What we want model to predict

adversarial_image = attacker.attack(clean_image, target_label)

# Verify attack

adversarial_pred = model(adversarial_image).argmax()
l2_distance = torch.norm(adversarial_image - clean_image).item()

print(f"Original: {original_pred}, Adversarial: {adversarial_pred}, L2: {l2_distance:.4f}")
```

### **1. C&W L0 Attack**
Minimizes number of pixels changed (sparse perturbation)

```python
# L0: Count number of pixels changed
```

Minimizes maximum per-pixel change

```python

loss = ||δ||_∞ + c × f(x + δ)

```

### **3. C&W with Different Confidence Levels**

```python
κ = 0:   Barely misclassify (logit difference ≈ 0)
κ = 5:   Medium confidence (5-unit margin)
κ = 20:  High confidence (20-unit margin)
κ = 40:  Very high confidence
```

# **3. PGD - Projected Gradient Descent**

## **Overview**

**Projected Gradient Descent (PGD)** (Madry et al., 2018) is an **iterative adversarial attack** that combines multi-step optimization with constraint projection. It's considered the **strongest first-order attack** and is the gold standard for adversarial training.

**Key Characteristics:**
- **Power**: Very effective (stronger than FGSM)
- **Speed**: Faster than C&W, slower than FGSM
- **Robustness**: More robust to defenses than FGSM
- **Standard**: De facto standard for adversarial training

**Core Idea:** Take multiple small steps in the gradient direction, projecting back to a valid constraint set after each step.

---

## **Mathematical Foundation**

### **The Optimization Problem:**

```
maximize L(θ, x + δ, y) subject to ||δ||_∞ ≤ ε

Where:
- L = loss function (e.g., cross-entropy)
- θ = model parameters
- δ = perturbation
- ε = maximum perturbation per pixel
- ||·||_∞ = L∞ norm (maximum absolute value)
```

### **The PGD Algorithm:**

```
x_0 = x + uniform_random(-ε, ε)  # Random initialization

for t = 0 to T-1:
    # Gradient ascent step
    x_{t+1} = x_t + α × sign(∇_x L(θ, x_t, y))
    
    # Project back to ε-ball around original x
    x_{t+1} = Π_{x + S}(x_{t+1})
    
    # Clip to valid pixel range
    x_{t+1} = clip(x_{t+1}, 0, 1)

return x_T

Where:
- T = number of iterations
- α = step size per iteration
- Π_{x + S} = projection operator onto ε-ball
- S = {δ : ||δ||_∞ ≤ ε}
```

---

## **How PGD Works: Step-by-Step**

### **Detailed Walkthrough:**

```
Setup:
- Original image: x (panda, 224×224×3)
- Model: CNN classifier
- ε = 8/255 (maximum perturbation per pixel)
- α = 2/255 (step size)
- T = 40 (number of iterations)

Step 1: Random Initialization
-------
Why? Makes attack stronger by exploring different starting points.

x_0 = x + uniform_random[-ε, ε]

Example:
x[0,0,0] = 0.523
random[0,0,0] = +0.015  (within [-8/255, +8/255])
x_0[0,0,0] = 0.538

Step 2: Iteration 1
-------
2a. Compute Gradient:
    Forward pass: x_0 → Model → Loss
    Backward pass: ∇_x L(θ, x_0, y)
    
    gradient[0,0,0] = +0.0031

2b. Gradient Ascent Step:
    x_1' = x_0 + α × sign(gradient)
    x_1'[0,0,0] = 0.538 + (2/255) × sign(+0.0031)
                = 0.538 + 0.0078 × (+1)
                = 0.546

2c. Project to ε-ball:
    Ensure ||x_1' - x||_∞ ≤ ε
    
    Compute: perturbation = x_1' - x
    perturbation[0,0,0] = 0.546 - 0.523 = 0.023
    
    Is 0.023 ≤ 8/255 (0.031)? Yes ✓
    So x_1[0,0,0] = 0.546
    
    If it exceeded ε:
    x_1[0,0,0] = clip(x_1'[0,0,0], x[0,0,0] - ε, x[0,0,0] + ε)

2d. Clip to Valid Range:
    x_1[0,0,0] = clip(0.546, 0, 1) = 0.546 ✓

Step 3: Iterations 2-40
-------
Repeat Steps 2a-2d for each iteration, gradually moving toward
adversarial example that maximizes loss.

Each iteration:
- Computes fresh gradient (more accurate than single FGSM step)
- Takes small step α (prevents overshooting)
- Projects back (stays within ε-ball)

Step 4: Final Result
-------
After 40 iterations:
x_40 = adversarial example

Test:
Model(x) → "Panda" (99.3%)
Model(x_40) → "Gibbon" (98.7%)
||x_40 - x||_∞ = 8/255 (at boundary)
```

---

## **Key Components Explained**

### **1. Random Initialization**

```python
# Why random start?
x_0 = x + uniform_random(-ε, +ε)
```

**Benefits:**
- **Escapes local optima**: Different starting points explore different paths
- **More robust**: Averages over multiple restarts
- **Stronger attack**: Finds better adversarial examples

**PGD vs I-FGSM:**
- **I-FGSM**: Starts from clean image (x_0 = x)
- **PGD**: Starts from random point in ε-ball

### **2. Projection Operator Π**

```python
def project_l_infinity(x_adv, x_clean, epsilon):
    """
    Project x_adv onto epsilon-ball around x_clean
    """
    # Compute perturbation
    delta = x_adv - x_clean
    
    # Clip to [-ε, +ε] per pixel
    delta = torch.clamp(delta, -epsilon, epsilon)
    
    # Return projected point
    return x_clean + delta
```

**Geometric Interpretation:**
```
ε-ball in L∞ norm = hypercube

If x_adv goes outside the cube:
    Pull it back to nearest point on cube surface
```

### **3. Step Size α**

**Rule of thumb:**
```
α = ε / T × 1.25

Example:
ε = 8/255, T = 40
α = (8/255) / 40 × 1.25 ≈ 2.5/255
```

**Trade-off:**
- **Large α**: Faster convergence, risk of oscillation
- **Small α**: More stable, may need more iterations

## **PGD: Code Implementation**

### **PyTorch Implementation:**

```python
import torch
import torch.nn as nn

def pgd_attack(model, images, labels, epsilon=8/255, alpha=2/255, 
               num_iter=40, random_start=True, targeted=False, target_labels=None):
    """
    PGD (Projected Gradient Descent) Attack
    
    Args:
        model: Target neural network
        images: Clean images (tensor)
        labels: True labels (for untargeted) or target labels (for targeted)
        epsilon: Maximum perturbation (L∞ norm)
        alpha: Step size per iteration
        num_iter: Number of iterations
        random_start: Whether to start from random point in ε-ball
        targeted: If True, perform targeted attack
        target_labels: Target class for targeted attack
    
    Returns:
        Adversarial examples (tensor)
    """
    
    # Make copy to avoid modifying original
    images = images.clone().detach()
    labels = labels.clone().detach()
    
    # Loss function
    criterion = nn.CrossEntropyLoss()
    
    # Random initialization
    if random_start:
        # Start from random point in ε-ball
        delta = torch.empty_like(images).uniform_(-epsilon, epsilon)
        adv_images = torch.clamp(images + delta, 0, 1).detach()
    else:
        # Start from clean image
        adv_images = images.clone().detach()
    
    # PGD iterations
    for iteration in range(num_iter):
        adv_images.requires_grad = True
        
        # Forward pass
        outputs = model(adv_images)
        
        # Compute loss
        if targeted:
            # Targeted: minimize loss for target class
            loss = -criterion(outputs, target_labels)
        else:
            # Untargeted: maximize loss for true class
            loss = criterion(outputs, labels)
        
        # Backward pass
        model.zero_grad()
        loss.backward()
        
        # Get gradient
        grad = adv_images.grad
        
        # Gradient ascent step
        with torch.no_grad():
            # For targeted: gradient descent (negative sign)
            # For untargeted: gradient ascent (positive sign)
            if targeted:
                adv_images = adv_images - alpha * grad.sign()
            else:
                adv_images = adv_images + alpha * grad.sign()
            
            # Project to ε-ball (L∞)
            perturbation = torch.clamp(adv_images - images, -epsilon, epsilon)
            adv_images = images + perturbation
            
            # Clip to valid pixel range [0, 1]
            adv_images = torch.clamp(adv_images, 0, 1).detach()
    
    return adv_images

# Usage Example 1: Untargeted Attack
model.eval()

true_labels = ...   # Shape: [batch_size]

    images=clean_images,
    labels=true_labels,
    epsilon=8/255,
    alpha=2/255,
    num_iter=40,
    random_start=True,
    targeted=False
)

# Usage Example 2: Targeted Attack

    target_labels=target_labels
)

# Usage Example 3: Multiple Random Restarts (stronger attack)
def pgd_with_restarts(model, images, labels, epsilon, alpha, num_iter, num_restarts=10):
    """
    PGD with multiple random restarts (finds best attack)
    """
    best_adversarial = images.clone()
    max_loss = torch.zeros(images.shape[0])
    
    for restart in range(num_restarts):
        # Run PGD from random start
        adv_images = pgd_attack(
            model, images, labels, epsilon, alpha, num_iter, 
            random_start=True, targeted=False
        )
        
        # Evaluate loss
        outputs = model(adv_images)
        loss = nn.CrossEntropyLoss(reduction='none')(outputs, labels)
        
        # Keep best adversarial examples (highest loss)
        for i in range(images.shape[0]):
            if loss[i] > max_loss[i]:
                max_loss[i] = loss[i]
                best_adversarial[i] = adv_images[i]
    
    return best_adversarial

)
```

## **PGD Variants and Extensions**

### **1. PGD with L2 Norm**

Instead of L∞ (max per-pixel change), constrain L2 (Euclidean distance):

```python
def pgd_l2_attack(model, images, labels, epsilon, alpha, num_iter):
    """
    PGD with L2 constraint
    """
    adv_images = images.clone()
    
    for i in range(num_iter):
        adv_images.requires_grad = True
        loss = nn.CrossEntropyLoss()(model(adv_images), labels)
        
        model.zero_grad()
        loss.backward()
        
        with torch.no_grad():
            # L2 normalized gradient
            grad = adv_images.grad
            grad_norm = torch.norm(grad.view(grad.shape[0], -1), p=2, dim=1)
            grad_normalized = grad / grad_norm.view(-1, 1, 1, 1)
            
            # Gradient ascent
            adv_images = adv_images + alpha * grad_normalized
            
            # Project to L2 ball
            perturbation = adv_images - images
            perturbation_norm = torch.norm(perturbation.view(perturbation.shape[0], -1), p=2, dim=1)
            scaling_factor = torch.clamp(perturbation_norm / epsilon, min=1.0)
            perturbation = perturbation / scaling_factor.view(-1, 1, 1, 1)
            
            adv_images = images + perturbation
            adv_images = torch.clamp(adv_images, 0, 1).detach()
    
    return adv_images
```

```python
def momentum_pgd(model, images, labels, epsilon, alpha, num_iter, decay_factor=1.0):
    """
    PGD with momentum
    """
    adv_images = images.clone()
    momentum = torch.zeros_like(images)
    
    for i in range(num_iter):
        adv_images.requires_grad = True
        loss = nn.CrossEntropyLoss()(model(adv_images), labels)
        
        model.zero_grad()
        loss.backward()
        
        with torch.no_grad():
            grad = adv_images.grad
            
            # Update momentum
            momentum = decay_factor * momentum + grad / torch.norm(grad, p=1)
            
            # Gradient ascent with momentum
            adv_images = adv_images + alpha * momentum.sign()
            
            # Project and clip
            perturbation = torch.clamp(adv_images - images, -epsilon, epsilon)
            adv_images = torch.clamp(images + perturbation, 0, 1).detach()
    
    return adv_images
```

```python
def comprehensive_evaluation(model, test_loader):
    """
    Evaluate model against multiple attacks
    """
    results = {
        'clean_accuracy': 0,
        'fgsm_accuracy': 0,
        'pgd_accuracy': 0,
        'cw_accuracy': 0
    }
    
    for images, labels in test_loader:
        # Clean accuracy
        clean_pred = model(images).argmax(dim=1)
        results['clean_accuracy'] += (clean_pred == labels).sum().item()
        
        # FGSM attack
        fgsm_adv = fgsm_attack(model, images, labels, epsilon=8/255)
        fgsm_pred = model(fgsm_adv).argmax(dim=1)
        results['fgsm_accuracy'] += (fgsm_pred == labels).sum().item()
        
        # PGD attack
        pgd_adv = pgd_attack(model, images, labels, epsilon=8/255, alpha=2/255, num_iter=40)
        pgd_pred = model(pgd_adv).argmax(dim=1)
        results['pgd_accuracy'] += (pgd_pred == labels).sum().item()
        
        # C&W attack (subset, expensive)
        cw_attacker = CWL2Attack(model)
        cw_adv = cw_attacker.attack(images[:10], labels[:10])  # Only first 10
        cw_pred = model(cw_adv).argmax(dim=1)
        results['cw_accuracy'] += (cw_pred == labels[:10]).sum().item()
    
    return results
```

# **Defenses Against These Attacks**

## **1. Adversarial Training**

Train model on adversarial examples:

```python
def adversarial_training(model, train_loader, epochs, epsilon=8/255):
    """
    Train model with adversarial examples
    """
    optimizer = optim.Adam(model.parameters())
    criterion = nn.CrossEntropyLoss()
    
    for epoch in range(epochs):
        for images, labels in train_loader:
            # Generate adversarial examples (typically PGD)
            adv_images = pgd_attack(model, images, labels, epsilon)
            
            # Train on mix of clean and adversarial
            all_images = torch.cat([images, adv_images])
            all_labels = torch.cat([labels, labels])
            
            optimizer.zero_grad()
            outputs = model(all_images)
            loss = criterion(outputs, all_labels)
            loss.backward()
            optimizer.step()
```

## **Key Takeaways**

1. **FGSM**: Fast and simple, good starting point
2. **C&W**: Most powerful, use for security-critical evaluation
3. **PGD**: Gold standard for adversarial training and benchmarking

4. **Attack Selection Hierarchy**:
   - **Quick test**: FGSM
   - **Standard eval**: PGD (40 iterations, 10 restarts)
   - **Final verification**: C&W

5. **Defense is Hard**: No perfect defense exists yet
   - Adversarial training reduces accuracy
   - Trade-off between clean and robust accuracy
   - Arms race continues
